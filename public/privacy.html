Privacy policy · Free beta

Your context should stay under your control.

The installable Context MRI Codex plugin runs locally, makes no network requests, and retains no supplied context. The public demonstration keeps uploaded files in browser memory for the current session.

What the local plugin processes

The plugin processes only the task, evaluator selection, and context content explicitly supplied to one of its tools. It does not automatically read your repository, Codex history, account, environment variables, or other files.

Storage and transmission

  • The local stdio plugin does not send context to Context MRI or a third-party server.
  • It does not create an account, require an API key, use analytics, or retain tool inputs after the process ends.
  • Diagnostic output omits raw context content and returns findings, hashes, scores, traces, and guard metadata.

These statements describe the Context MRI plugin boundary only. Codex and model execution follow the user's Codex/OpenAI configuration and may use OpenAI services.

Public web demonstration

The hosted demonstration uses deterministic bundled fixture evidence. Files added through the interface are held in browser memory for the current session and are not persisted by the public demo. Do not submit secrets or regulated data to any beta software.

Future hosted service

No paid or hosted context-analysis service is enabled in this competition build. If one is introduced later, this policy will be updated before launch with clear retention, deletion, tenant-isolation, and security terms.

Questions and changes

Questions or vulnerability reports can be opened through the public GitHub repository. Material policy changes will be dated on this page.
